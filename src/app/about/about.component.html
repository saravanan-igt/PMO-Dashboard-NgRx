Best Practices

  • Application is designed with Atomic design pattern by creating small reusable components.
  • Services has been isolated from the components for invoking APIs.
  • NgRx state management has been utilized to share data across the application. This also reduced multiple API calls.
  • Backend request will be sent only once after login.
  • CDK Virtual Scroll has been used which enables a performant way to simulate all items being rendered by making the height of the container element the same as the height of the total number of elements to be rendered, and then only rendering the items in view. 
  • ng build --prod
    should be used for Production build.
  • Used GZip compression at the backend which reduces response data size from 400kb to 34kb and also reduces the size of .js, .css, image files by 70%.
  • Cached the static files like .js, .css, images. Caching will reduce the loading time of these files eventually increases performance.
